# LuSplit UX Principles

Version 1.0 - Product Direction

## Core Experience

LuSplit is a calm utility for shared expenses. UX should reduce tension and increase clarity.

Design for:

- Relief
- Clarity
- Calm resolution

## Visual Principles

- Soft visual language
- Rounded components
- Airy spacing
- Low visual noise
- Readability-first hierarchy

## Interaction Principles

- Fast, subtle micro-animations (`120–180ms`)
- Ease-out cubic motion
- No bouncing effects
- Keep state changes legible and reassuring

## Language Principles

- Use friendly, direct copy
- Prefer everyday terms (`Owes`, `Settle up`, `Expense`, `Friends`)
- Avoid jargon-heavy finance language

## Mobile Direction

### Home Screen

- Header with group name and total balance summary
- Body with expense list
- Footer with primary CTA: “Add expense”
- Bottom tab navigation (max 4 tabs)

### Add Expense Screen

- Simple form
- Large inputs
- People selection chips
- Equal split default

## Web Direction

- Centered layout
- Maximum content width: `1024px`
- Generous padding
- Optional sidebar: groups and settings
- Prioritize readability

## Accessibility Principles

- Minimum touch target: `44px`
- Fully responsive text scaling
- Dark mode is first-class (not an afterthought)
- Maintain at least `4.5:1` contrast
