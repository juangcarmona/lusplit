# Export Format

Exports must be complete and re-importable. Available from Trip Details → "Export trip".

## JSON snapshot

**File:** `{trip-slug}-{date}.snapshot.json`  
**MIME:** `application/json`  
**Purpose:** Canonical full-fidelity export, designed for future import.

Fields:
- `schemaVersion`: integer (currently 1)
- `tripName`: string (from app metadata)
- `exportedAt`: ISO-8601 timestamp
- `group`: `{ id, currency, closed }`
- `participants`: array with id, name, economicUnitId, consumptionCategory
- `economicUnits`: array with id, ownerParticipantId, name
- `expenses`: array with id, title, paidByParticipantId, **amountMinor** (integer), date, splitDefinition, notes
- `transfers`: array with id, fromParticipantId, toParticipantId, **amountMinor** (integer), date, type, note

All amounts in minor units (integers) for lossless roundtrip.

## CSV bundle

**File:** `{trip-slug}-{date}-export.zip`  
**MIME:** `application/zip`  
**Purpose:** Spreadsheet-friendly analysis. UTF-8 with BOM.

Contains:
- `expenses.csv` — id, title, amount (decimal), currency, paid_by, date, notes
- `members.csv` — id, name, household, consumption_category, custom_weight
- `transfers.csv` — id, from, to, amount (decimal), currency, date, type, note
- `balances.csv` — person, balance (decimal), currency, direction

Money in CSV is formatted as a decimal (e.g. `10.00`) derived from minor units.

## PDF summary

**File:** `{trip-slug}-{date}-summary.pdf`  
**MIME:** `application/pdf`  
**Purpose:** Human-readable sharing (email, print).

Sections:
- Trip name + export date
- Friends list
- Balances / settle-up plan
- Expenses list (title, paid by, amount, date)
- Footer: "Generated by LuSplit"

Generated as PDF/1.4 with Helvetica (standard Type 1 font, no embedding required).

## File naming

`{slug}-{yyyy-MM-dd}[.snapshot.json | -export.zip | -summary.pdf]`

Slug is the trip name lowercased, with non-alphanumeric characters replaced by `-`, max 40 chars.
